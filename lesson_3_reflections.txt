When would you want to create a remote repository rather than keeping all your work local?
  Um repositorio remoto é melhor do que manter o trabalho local quando você deseja que outras pessoas
trabalhem no mesmo projeto ou quando voce deseja compartihar o seu projeto com outros.

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?
  Sempre vou querer fazer "pull" manualmente, pois assim terei controle de quando quero atualizar com o 
repositorio remoto. Ou seja, quero controlar o momento certo para trazer localmente alguma alteração efetuada
por outrem.

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?
  Forks: somente para repositorio remoto
  Clones: somente para repositorio local
  Branch: somente no mesmo repositorio

How would you collaborate without using Git or GitHub?  What would be easier,
and what would be harder?

When would you want to make changes in a separate branch rather than directly in
master?  What benefits does each approach have?
  Fazer alterações em um branch separado permite que tenhamos melhor controle sobre o impacto dessa alteração
principalmente quando outros estão alterando o mesmo codigo.
